# masterdata
Binary evidence storage for the MK2 investigation framework.

## Structure

| Directory | Contents | Source |
|-----------|----------|--------|
| `images/evidence/` | Investigation screenshots and photos | All repos |
| `images/bingo/` | BINGO evidence images | Claude-MKIIupd |
| `images/thebullet/` | THEBULLETFROMSMOKINGUN images | Claude-MKIIupd |
| `images/linux-logs/` | Linux log screenshots | DATABASE |
| `images/smashers/` | Smashers-HQ investigation images | Smashers-HQ |
| `images/malware/` | Malware investigation images | malware-invasion |
| `images/database/` | DATABASE investigation images | DATABASE |
| `videos/` | Video evidence (.mov) | Claude-MKIIupd |
| `journals/` | systemd journal binary files | Claude-MKIIupd |
| `logs-binary/` | EVTX and large log files | malware-invasion, DATABASE |
| `pdfs/` | PDF reports and documents | Threat-2, Smashers-HQ |
| `documents/` | .docx files | Smashers-HQ |
| `exports/` | Large data exports (.json, .csv) | Smashers-HQ, Threat-2 |

## Companion Repo

Text content, reports, and analysis are in [masterhq](https://github.com/Smooth511/masterhq).

## Manifest

See `masterdata-manifest.json` in masterhq for a complete file-by-file listing of what goes where.
